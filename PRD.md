# 产品需求文档（Product Requirements Document）

## 1. 产品概述
- **产品名称**：拼好班
- **产品定位**：创新型灵活用工平台，通过"拼团"模式连接自由职业者与项目团队
- **目标用户**：自由职业者（设计师、开发者、营销人员等）、项目发起方（企业、创业团队、个人）
- **核心问题**：解决自由职业者收入不稳定、项目机会少、时间利用不充分的问题；解决项目方组建团队成本高、人才匹配难的问题
- **产品愿景**：成为中国领先的灵活用工协作平台，让每个人都能通过"拼工作"实现时间价值最大化和收入增长

## 2. 市场与用户分析

### 2.1 市场分析
- **市场现状**：
  - 自COVID-19疫情以来，全球自由职业经济持续繁荣，越来越多人选择灵活就业
  - 中国灵活用工市场快速增长，远程工作和零工经济成为主流趋势
  - 现有平台主要采用"1对1"匹配模式（雇主直接雇佣单个自由职业者）

- **竞品分析**：
  - **Upwork**：综合型平台，竞标系统，佣金5-20%，缺乏团队协作功能
  - **Fiverr**：基于Gig的服务市场，2026年通过AI工具持续增长，但仍是单人服务模式
  - **Toptal**：高端人才平台，严格筛选，定位顶级专业人士，门槛高
  - **国内平台**：猪八戒、自由人等，主要是任务外包模式，缺乏团队组建功能

- **市场机会**：
  - 现有平台均未提供"团队拼组"功能，存在明显的差异化空间
  - 自由职业者希望同时参与多个项目以提高收入和时间利用率
  - 项目方需要更灵活、成本更低的团队组建方式

- **商业价值**：
  - 平台佣金收入（建议低于传统平台的5-15%）
  - 会员增值服务（VIP推荐、优先匹配）
  - 企业服务（团队管理工具、数据分析）

### 2.2 用户分析
- **用户画像**：
  - **自由职业者**：25-40岁，具备专业技能（设计、开发、营销、文案等），希望增加收入、灵活安排时间
  - **项目发起方**：中小企业、创业团队、个人创业者，需要快速组建项目团队，控制成本

- **用户痛点**：
  - **自由职业者痛点**：
    - 项目来源不稳定，收入波动大
    - 单打独斗，缺乏团队支持和归属感
    - 闲暇时间无法充分利用，机会成本高
    - 难以同时管理多个项目机会
  - **项目方痛点**：
    - 组建完整团队成本高、周期长
    - 难以找到技能互补的人才组合
    - 管理分散的自由职业者困难
    - 缺乏信任机制，合作风险高

- **使用场景**：
  - **场景1 - 自由职业者找项目**：设计师小李在下班后有3小时空闲时间，打开"拼好班"浏览正在招募的设计项目，发现一个电商项目缺UI设计师，团队已有产品经理和前端开发，申请加入并通过审核，开始兼职工作
  - **场景2 - 多项目并行**：开发者小王同时参与3个项目团队，周一到周三为A项目工作，周四周五为B项目工作，周末为C项目工作，充分利用时间，月收入提升50%
  - **场景3 - 项目方组队**：创业者老张需要开发一个小程序，在"拼好班"发布项目需求（需要1名产品经理、2名开发、1名设计师），平台推荐合适人选，快速组建团队，成本比传统外包低30%

## 3. 功能架构
- **产品架构**：
  - **用户层**：注册登录、个人主页、技能认证、信用体系
  - **项目层**：项目发布、项目浏览、搜索筛选、项目详情
  - **匹配层**：申请加入、团队组建、智能推荐、审核管理
  - **协作层**：消息沟通、进度管理、文件共享、任务分配
  - **交易层**：支付结算、收入统计、发票管理

- **页面清单**：
  - 首页（项目推荐流）
  - 项目列表页（搜索筛选）
  - 项目详情页（团队配置、申请入口）
  - 我的项目（参与中、已完成）
  - 个人主页（技能展示、作品集、评价）
  - 消息中心（系统通知、团队沟通）
  - 发布项目页（项目方）
  - 团队管理页（项目方）

- **信息架构**：
  - 底部Tab导航：首页、项目、消息、我的
  - 首页：推荐项目流 + 快速搜索
  - 项目页：我参与的项目列表
  - 消息页：系统通知 + 团队消息
  - 我的：个人信息 + 设置

### 关键用户流程
- **主要路径（自由职业者）**：
  1. 注册登录 → 完善个人信息和技能认证
  2. 浏览首页推荐项目 或 搜索感兴趣的项目
  3. 查看项目详情，了解团队配置和要求
  4. 申请加入团队，填写申请理由
  5. 等待项目方审核，收到通过通知
  6. 确认参与项目，加入团队
  7. 通过消息系统与团队沟通，推进项目
  8. 项目完成后互相评价，获得收入和信用分
  9. 继续浏览新项目，同时参与多个团队

- **主要路径（项目方）**：
  1. 注册登录 → 完善企业/个人信息
  2. 发布项目，填写项目信息和团队配置
  3. 等待自由职业者申请，查看申请者资料
  4. 审核申请者，选择合适的团队成员
  5. 团队组建完成，项目启动
  6. 通过消息系统与团队沟通，管理项目进度
  7. 项目完成后评价团队成员，完成结算
  8. 发布新项目或继续与优秀成员合作

## 5. 产品约束与规范
- **平台要求**：
  - iOS 14.0及以上版本
  - 支持iPhone和iPad
  - 适配深色模式

- **性能要求**：
  - 首页加载时间<3秒
  - 搜索响应时间<2秒
  - 消息推送延迟<30秒
  - 图片加载采用渐进式加载
  - 支持离线浏览已加载内容

- **技术约束**：
  - 使用Swift开发
  - 采用RESTful API架构
  - 数据传输使用HTTPS加密
  - 支持推送通知（APNs）
  - 图片存储使用CDN加速

- **内容规范**：
  - 项目名称：2-50字
  - 项目描述：10-2000字
  - 技能标签：从预设标签库选择
  - 用户昵称：2-20字，不允许特殊字符
  - 头像：支持JPG/PNG，最大5MB
  - 作品集：支持图片、PDF、链接

- **合规要求**：
  - 用户必须完成实名认证
  - 遵守《网络安全法》和《个人信息保护法》
  - 敏感信息加密存储
  - 提供用户协议和隐私政策
  - 支持用户注销账号和数据删除
  - 建立内容审核机制，防止违规信息

## 6. 待澄清项
1. **项目类型定位**：是否限定特定领域（如仅技术开发、设计），还是全品类开放（包括营销、文案、咨询等）？建议初期聚焦1-2个垂直领域，验证模式后再扩展。

2. **盈利模式细节**：
   - 佣金比例？建议5-10%，低于传统平台以吸引用户
   - 是否收取项目发布费？
   - 会员体系设计？（如VIP会员享受优先推荐、更多申请次数）
   - 增值服务有哪些？（如背景调查、合同模板、保险服务）

3. **团队规模限制**：
   - 每个项目最少/最多多少人？建议2-10人
   - 是否允许1人项目（退化为传统模式）？

4. **项目周期与类型**：
   - 支持的项目周期范围？建议1周-6个月
   - 是否支持长期合作（如按月续约）？
   - 是否支持紧急项目（如24小时内启动）？

5. **支付与结算**：
   - MVP阶段是否包含支付功能？建议初期线下结算，降低开发成本
   - 资金托管方案？需要支付牌照或第三方托管
   - 分账规则？按角色、工作量还是固定比例？

6. **信用体系设计**：
   - 初始信用分如何设定？
   - 信用分计算公式？（完成率、评价、活跃度等权重）
   - 低信用分用户的限制措施？

7. **地域限制**：
   - 是否支持跨地域远程协作？
   - 是否需要标注项目地点？
   - 是否支持线下见面的项目？

8. **审核机制**：
   - 项目发布是否需要平台审核？
   - 用户资料审核标准？
   - 争议处理流程？
